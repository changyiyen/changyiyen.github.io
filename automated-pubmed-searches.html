Automated PubMed searches

This has been done probably countless times before, but this is my (very rudimentary and totally insecure) take on automating PubMed searches, using the E-utilities API and wget/cURL/fetch. Note that it's probably a better idea to just use the EDirect utilities available for your platform, since they can pipe data to each other without ad-hoc processing, and are designed to respect NCBI-imposed lookup rate limits. I wrote this partly as a very small exercise in sh scripting.

This script relies on XMLStarlet (as the xml command) for parsing search results. It fetches the relevant article abstracts into a single file, then compresses it. It is meant to be run as a cron job, probably once daily.

If you plan on using this script, please be sure to: 1. change the result directory to whatever applies to your system, and 2. change the search query terms as needed.

Update 2022-08-03: One of my colleagues asked me whether a version of this script is available for Windows, so I reimplemented it in PowerShell. Since PowerShell can parse XML natively via the Select-Xml cmdlet, this script now has no dependencies.
